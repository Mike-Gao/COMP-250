Introduction

This course is an introduction to several core topics in computer science, namely data structures and algorithms, and object oriented design. For the data structures and algorithms component, you will learn about various types of lists (arrays, linked lists, stacks, queues), trees, and graphs. You will also learn some basic algorithms that use these data structures, for example, to do searching and sorting, and how to analyze such algorithms in terms of the amount of computation that is required to carry them out. These analysis tools will be used heavily in subsequent data structures and algorithms courses (COMP 251, 360) and are a central part of computer science. A second significant part of the course is a deeper appreciation of object oriented design from what you had in COMP 202. You will learn how classes can be organized into hierarchies, and how variables and methods defined in the classes of the hierarchy are related to each other. These relationships, known as inheritance and polymorphism, will developed more fully in subsequent courses that follow the software stream of the CS curriculum.

Prerequisites

The prerequisite for this course is COMP 202 Introduction to Computing 1 or any one semester course on programming in a high level language such as C or Java. Because the majority of students in COMP 250 have taken COMP 202 (or equivalent), most students will know the basics of Java already. As such, I will not be reviewing basic Java.

If you have not taken COMP 202 (or an equivalent Java course), then you should see a recent COMP 202 Course Outline to make sure you have the appropriate background for COMP 250. At a bare minimum, you should have experience with programming in a high level language and you should have written several programs that use variables, expressions, conditional statements (if-then-else, switch), loops (while, for), arrays, pointers (references in Java), basic I/O (keyboard inputs or reading from a file, writing to console or to a file).

COMP 202 uses Java which is an object oriented language, and so students who have taken 202 will also be familiar with basic concepts in object oriented programming (OOP). Those of you who have only programmed in C (but not C++) will be unfamiliar with the basic concepts of OOP such as objects and classes. This applies, in particular, to some Fac. Eng. students who have taken COMP 208. If you want to take COMP 250, you will need to do some work in the first few weeks of the semester to catch up. In particular, I strongly suggest that you sign out from the library or buy a basic book on Java programming and read the parts of it that are covered in COMP 202 and work through some of the programming examples. Two very good introductory texts for Java are:
  • D. Liang, ``Introduction to Java Programming,'' Brief Edition, Prentice Hall.
  • Lewis and Loftus, ``Java software solutions,'' Addison-Wesley.
There is also plenty of good material online, for example,
  • a book ``How to think like a computer scientist'' . The current (Aug. 2012) version of that book covers roughly the same material as COMP 202.
  • Java website  + tutorials + API ( SE 6)
I have prepared a document with Java terms that you should be familiar with, after reviewing basic Java . Once you are familiar with most of the terms in that document and have written some basic Java programs yourself, you will be ready for COMP 250.

Evaluation

Your final grade will be calculated using the following percentage breakdown.
  • Four Assignments (40 % total)
    • A1 to be posted in mid-September
    • A2 to be posted in early October
    • A3 to be posted in late October
    • A4 to be posted in mid-November
    You will be given at least 10 days to complete each assignment.

  • Two in-class Quizzes (20 % = 2 x 10 %)}
    • Quiz 1 in early October
    • Quiz 2 in early November

  • Final Exam (40 % or 60 %) held during Final Examination Period.
If your percentage grade on the Final Exam is greater than your percentage grade on the two Quizzes, then your Final Exam grade will count for 60 % of your final grade and the two Quizzes will count for 0%. This policy is intended to allow some flexibility, for example, if you are unable to attend one of the quizzes because you are sick or you have some other obligation.

Note that while there is some flexibility in the exams, there is no flexibility with the assignments. If you don't do an assignment, then you get 0.

In accord with McGill University's Charter of Students' Rights, students in this course have the right to submit in English or in French any written work that is to be graded.

Policy on collaboration

We greatly encourage you to discuss the assignment problems with each other. However, this discussion must not go so far that you are revealing the solutions to each other. And it certainly should not go so far that you are sharing code. We realize there is sometimes a fine line between giving hints and revealing solutions, so we ask you try to follow a simple guideline: for any discussion you have about an assignment, you should be comfortable if anyone else including the instructor were listening in.
